localgcp

{ the unified GCP emulator }

One binary, four services, zero cloud bills. Develop against Cloud Storage, Pub/Sub, Secret Manager, and Firestore without leaving localhost.

localgcp
$ go install github.com/slokam-ai/localgcp@latest $ localgcp up Starting localgcp... Cloud Storage :4443 Pub/Sub :8085 Secret Manager :8086 Firestore :8088 localgcp is ready. Press Ctrl+C to stop.

What you get

Four services. One process.

Your GCP client libraries already support emulator host env vars. Point them at localhost and your existing code works with zero changes.

Cloud Storage
REST · :4443

Bucket and object CRUD. Simple, multipart, and resumable uploads. Prefix + delimiter listing. JSON and XML API paths.

Pub/Sub
gRPC · :8085

Topics, subscriptions, publish, pull, and StreamingPull. Fan-out delivery. Ack deadlines and redelivery.

Secret Manager
gRPC · :8086

Secrets with versioning. Enable, disable, destroy states. Access by version number or "latest" alias.

Firestore
gRPC · :8088

Document CRUD, structured queries with equality and range filters. OrderBy, limit, offset. Transactions with commit and rollback.

Get started

Three commands to local GCP

01

Install

Single binary. No Docker, no JVM, no runtime dependencies.

$ go install github.com/slokam-ai/localgcp@latest
02

Start

All four services in the foreground. In-memory by default, or persistent with --data-dir.

$ localgcp up
03

Connect

Sets emulator host env vars. Your GCP client libraries do the rest.

$ eval $(localgcp env) $ go run ./your-app

Why this exists

Before and after

Without localgcp

  • Spin up real GCP dev projects
  • Pay cloud bills for test environments
  • Manage individual emulators per service
  • Different setup for each service
  • No Cloud Storage emulator from Google
  • Requires internet connection
  • Shared environments, stepping on each other

With localgcp

  • One binary, starts in milliseconds
  • Zero cloud bills for dev and CI
  • All services in one process
  • Unified CLI, consistent behavior
  • Cloud Storage included
  • Works fully offline
  • Each developer gets their own instance
{ }
AWS has LocalStack. GCP has had nothing equivalent. Fragmented emulators, inconsistent APIs, missing services. Until now.
